## 开发规范

### 代码风格
- 前端统一使用ES6语法
- TypeScript使用时需要详细注释
- 所有新函数和类必须有注释
- 数据变量声明也需要有注释 

### 技术选型
- 后端优先使用Node.js，大型项目可考虑Go/Java/Python/PHP/Rust/C++/C#
- HTTP客户端统一使用axios/fetch/jquery
- 鉴权优先使用JWT，其他方案需要详细文档说明

### 数据库配置
- 优先使用本地MySQL数据库
  - 连接信息：
    - 主机：localhost
    - 用户名：root
    - 密码：Wangjun@123
    - 端口：3306（默认）
  - 数据库命名规范：
    - 使用小写字母
    - 多个单词用下划线连接
    - 建议格式：项目名_环境（例：myapp_dev, myapp_prod）
  - 表设计规范：
    - 必须包含主键（推荐使用自增id）
    - 必须包含创建时间和更新时间字段
    - 表名使用单数形式
    - 字段名使用下划线命名法
  - 安全要求：
    - 生产环境必须修改默认密码
    - 定期备份数据库
    - 限制数据库远程访问
  - 性能优化：
    - 适当建立索引
    - 大表考虑分区
    - 定期维护和优化

### 兼容性要求
- Node.js ≥ 18
- Python ≥ 3.10
- 浏览器支持：最新两个版本的主流浏览器

### 项目结构规范
- 前后端都需要有Utils工具类库，避免重复造轮子
- 网络请求必须封装，使用Promise风格调用
- 项目必须包含完整的README.md，详细说明项目结构和使用方法
- 每次修改后及时更新文档，包括每个文件的用途说明


### 错误处理规范
- 统一的错误处理机制
- 区分用户错误和系统错误
- 记录关键错误日志


### 性能优化原则
- 前端：
  - 合理的代码分割和懒加载
  - 静态资源优化（压缩、CDN）
  - 减少HTTP请求
  - 使用合适的缓存策略
- 后端：
  - 合理使用数据库索引
  - 实现适当的缓存机制
  - API响应时间监控
  - 合理的数据分页

### 文档要求
-除过readme文档外，其他API文档、技术文档等都需要存放在统一的文档文件夹中
- 代码注释需要说明：
  - 功能描述
  - 参数说明
  - 返回值说明
  - 使用示例
  - 注意事项
- API文档需要及时更新
- 技术方案需要书面记录
- 重要决策需要文档化

## 开发流程
1. 理解需求，必要时提出疑问
2. 评估技术方案
3. 开发前检查相关文档
4. 遵循编码规范进行开发
6. 进行代码审查
7. 更新相关文档
8. 部署和验证
